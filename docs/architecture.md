## 数据流

### 交易创建流程

```
用户操作
    │
    ▼
┌─────────────┐
│  前端组件    │  ──(1)──>  调用 portfolioStore.createTrade()
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Store 层   │  ──(2)──>  调用 portfolioApi.createTrade()
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  API 客户端  │  ──(3)──>  HTTP POST /api/portfolio/trades
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Gin 路由   │  ──(4)──>  路由匹配，调用 CreateTrade handler
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Handler    │  ──(5)──>  参数校验、业务处理
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  GORM       │  ──(6)──>  数据库操作（事务）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  MySQL      │  ──(7)──>  数据持久化
└─────────────┘
```

### 数据查询流程

```
用户访问页面
    │
    ▼
┌─────────────┐
│  前端组件    │  ──(1)──>  调用 portfolioStore.fetchDashboard()
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Store 层   │  ──(2)──>  检查缓存，调用 API
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  API 客户端  │  ──(3)──>  HTTP GET /api/portfolio/dashboard
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Handler    │  ──(4)──>  查询数据库 + 调用外部 API
└──────┬──────┘
       │
       ├───(5a)──>  MySQL: 查询持仓和交易记录
       │
       └───(5b)──>  CoinCap API: 获取实时价格
       │
       ▼
┌─────────────┐
│  数据计算   │  ──(6)──>  计算盈亏、收益率等指标
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  JSON 响应  │  ──(7)──>  返回聚合数据
└─────────────┘
```

## 模块说明

### 1. 认证模块 (Auth)

**职责**: 用户注册、登录、Token 管理

**核心功能**:
- 用户注册/登录
- JWT Token 生成与验证
- 密码加密存储
- Token 刷新与失效

**数据模型**:
- User: 用户基本信息
- RefreshToken: 刷新令牌

### 2. 资产组合模块 (Portfolio)

**职责**: 资产管理、交易记录、盈亏计算

**核心功能**:
- 交易记录 CRUD
- 持仓计算（加权平均法）
- 盈亏指标计算
- 数据导出/导入

**数据模型**:
- Trade: 交易记录
- Holding: 持仓快照

**计算逻辑**: 详见 [calculations.md](./calculations.md)

### 3. 价格模块 (Prices)

**职责**: 加密货币价格获取

**核心功能**:
- 从 CoinCap API 获取实时价格
- 价格缓存
- 价格变化率计算

**外部依赖**:
- CoinCap API

### 4. 配置模块 (Config)

**职责**: 应用配置管理

**核心功能**:
- 多源配置加载（环境变量、配置文件）
- 配置验证
- 配置热更新

**配置项**: 详见 [configuration.md](./configuration.md)

## 安全设计

### 认证安全

- JWT Token 有效期 24 小时
- Refresh Token 机制
- 密码使用 bcrypt 加密
- 支持多设备登录管理

### 数据安全

- 数据库连接使用 TLS（可选）
- 敏感配置通过环境变量注入
- 配置文件不提交到代码仓库

### API 安全

- 所有业务接口需要认证
- 参数校验和 SQL 注入防护
- 限流保护（待实现）
