# 加密货币投资组合 - 指标计算说明

本文档详细说明系统中各种财务指标的计算方法。

## 一、基础概念

### 1.1 交易类型

| 类型 | 说明 | 对资产的影响 | 对USDT的影响 |
|------|------|-------------|-------------|
| **充值 (recharge)** | 向账户充值USDT | USDT持仓增加 | +Amount |
| **买入 (buy)** | 用USDT购买加密资产 | 加密资产增加 | -Total (总价) |
| **卖出 (sell)** | 出售加密资产获得USDT | 加密资产减少 | +Total (总价) |

### 1.2 核心公式

```
总价 (Total) = 数量 (Amount) × 单价 (Price)
```

## 二、成本计算

### 2.0 成本与成本价的区别

| 概念 | 定义 | 单位 | 示例 |
|------|------|------|------|
| **成本** | 持有某资产的总投入金额 | 金额（USDT） | 投入 $500,000 买入 BTC |
| **成本价** | 单位资产的平均买入价格 | 单价（USDT/个） | BTC 成本价为 $50,000/个 |

**联系：**
```
成本价 = 成本 / 持仓量
成本 = 成本价 × 持仓量
```

**举例说明：**
- 投入 $500,000 买入 10 个 BTC
- **成本** = $500,000（总投入）
- **成本价** = $50,000/个（单位成本）

---

### 2.1 成本价（平均成本）

成本价采用**加权平均法**计算：

```
成本价 = 当前成本 / 当前持仓量
```

**特殊情况处理：**

| 场景 | 成本价显示 | 说明 |
|------|-----------|------|
| 正常持仓 | 实际成本价 | 按公式计算 |
| 成本价 = 0 | "已回本" | 总投入与总退出持平 |
| 成本价 < 0 | "负成本" | 投资回报超过100%，持仓为"免费" |

### 2.2 成本动态计算

成本不是固定值，而是根据交易动态变化：

**买入时：**
```
新成本 = 原成本 + 本次买入总价
```

**卖出时：**
```
卖出比例 = 卖出数量 / 卖出前持仓量
回收成本 = 卖出前成本 × 卖出比例
新成本 = 原成本 - 回收成本
```

**示例：**
- 买入10个BTC，单价$50,000，成本 = $500,000
- 卖出6个BTC（卖出比例 = 6/10 = 60%）
- 回收成本 = $500,000 × 60% = $300,000
- 剩余成本 = $500,000 - $300,000 = $200,000（对应4个BTC）

## 三、盈亏指标

### 3.1 浮动盈亏（未实现盈亏）

反映当前持仓的账面盈亏，**尚未实际变现**。

```
浮动盈亏 = 当前市值 - 当前成本
         = (持仓量 × 当前价格) - 当前成本
```

**浮动盈亏率：**
```
浮动盈亏率 = (浮动盈亏 / 当前成本) × 100%
```

**示例：**
- 持仓：4个BTC
- 当前价格：$60,000
- 当前成本：$200,000
- 当前市值：4 × $60,000 = $240,000
- 浮动盈亏：$240,000 - $200,000 = **+$40,000**
- 浮动盈亏率：($40,000 / $200,000) × 100% = **+20%**

### 3.2 实现盈亏（已实现盈亏）

反映已经卖出部分的**实际盈亏**。

```
实现盈亏 = 卖出收入 - 回收成本
```

**实现盈亏率：**
```
实现盈亏率 = (实现盈亏 / 历史总投入) × 100%
```

**为什么用历史总投入而不是当前成本？**

实现盈亏率反映的是相对于**总投资**的回报率，而非剩余持仓的收益率。

**示例：**
- 历史总投入：$500,000（买入10个BTC）
- 卖出6个BTC，收入：$360,000（单价$60,000）
- 回收成本：$300,000
- 实现盈亏：$360,000 - $300,000 = **+$60,000**
- 实现盈亏率：($60,000 / $500,000) × 100% = **+12%**

### 3.3 总盈亏

```
总盈亏 = 浮动盈亏 + 实现盈亏
```

## 四、资产价值指标

### 4.1 加密资产市值

```
加密资产市值 = Σ(各币种持仓量 × 当前价格)
```

**注意：** 不包含USDT余额。

### 4.2 总资产价值

```
总资产价值 = 加密资产市值 + USDT余额
```

### 4.3 24小时价值变化

```
24小时变化率 = Σ(各币种市值 × 该币种24小时涨跌幅) / 总市值
```

采用**市值加权平均**计算。

## 五、综合示例

### 场景：BTC交易全流程

**第1步：充值**
- 充值 $500,000 USDT
- USDT余额：$500,000

**第2步：买入**
- 买入10个BTC，单价$50,000
- BTC持仓：10个
- BTC成本：$500,000
- USDT余额：$0

**第3步：价格上涨后部分卖出**
- BTC价格涨至$60,000
- 卖出6个BTC，收入$360,000
- BTC持仓：4个
- BTC成本：$200,000（回收$300,000）
- USDT余额：$360,000

**第4步：计算各项指标**

| 指标 | 计算过程 | 结果 |
|------|---------|------|
| BTC市值 | 4 × $60,000 | $240,000 |
| 总资产 | $240,000 + $360,000 | $600,000 |
| 浮动盈亏 | $240,000 - $200,000 | +$40,000 |
| 浮动盈亏率 | ($40,000 / $200,000) × 100% | +20% |
| 实现盈亏 | $360,000 - $300,000 | +$60,000 |
| 实现盈亏率 | ($60,000 / $500,000) × 100% | +12% |
| 总盈亏 | $40,000 + $60,000 | +$100,000 |

## 六、特殊情况处理

### 6.1 清仓资产（持仓为0）

- 仍显示在资产列表中（如果有实现盈亏）
- 市值、成本、浮动盈亏均为0
- 保留实现盈亏记录

### 6.2 负成本情况

当多次卖出后回收成本超过总投入时：

```
成本价 < 0
```

**含义：** 持仓已经是"免费"的，甚至整体已经盈利。

**显示：** "负成本"徽章 + 实际数值

### 6.3 零成本情况

```
成本价 = 0
```

**含义：** 总投入与总退出持平，投资已全部收回。

**显示：** "已回本"徽章 + 实际数值

## 七、数据结构

### 7.1 交易记录 (Trade)

```go
type Trade struct {
    ID        uint      // 交易ID
    UserID    uint      // 用户ID
    Symbol    string    // 币种（BTC、ETH、USDT等）
    Type      string    // 类型（buy、sell、recharge）
    Amount    float64   // 数量
    Price     float64   // 单价
    Total     float64   // 总价 = Amount × Price
    CreatedAt time.Time // 创建时间
}
```

### 7.2 持仓记录 (Holding)

```go
type Holding struct {
    ID        uint      // 持仓ID
    UserID    uint      // 用户ID
    Symbol    string    // 币种
    Amount    float64   // 当前持仓量
    CreatedAt time.Time // 创建时间
    UpdatedAt time.Time // 更新时间
}
```

**注意：** 不存储成本价，成本价通过交易记录实时计算。

## 八、计算流程

```
1. 获取所有交易记录
   ↓
2. 按时间顺序遍历交易
   - 买入：累加持仓和成本
   - 卖出：按比例回收成本，计算实现盈亏
   ↓
3. 获取当前价格
   ↓
4. 计算各资产指标
   - 市值、成本、浮动盈亏
   ↓
5. 汇总统计
   - 总市值、总盈亏、总实现盈亏
```

## 九、FAQ

**Q1: 为什么成本价会变化？**
A: 采用加权平均法，每次卖出都会按比例回收成本，导致剩余持仓的成本价变化。

**Q2: 实现盈亏和浮动盈亏有什么区别？**
A: 实现盈亏是已经卖出部分的实际盈亏（已变现），浮动盈亏是当前持仓的账面盈亏（未变现）。

**Q3: 为什么清仓后还能看到资产？**
A: 为了展示该资产的实现盈亏历史，帮助用户了解整体投资表现。

**Q4: 负成本是什么意思？**
A: 说明通过卖出已经回收了全部成本，甚至还赚了，剩余持仓是"免费"的。
